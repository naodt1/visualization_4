Text and Tree visualization and Colors

Analyzing document clusters

In this task you will create a system to analyze a clustering of short texts that describe action movies. It is your task to implement and coordinate both text and tree visualizations to find out about differences and similarities between these clusters.

The data analyst used a hierarchical cluster algorithm to generate the clusters from the movies' overview texts embeddings. In this process, initially, all items are in separate clusters. In each iteration step, the two most similar clusters are merged. The variable merge_tree contains a tree representation of the last 20 steps of the process. Each node contains 5 attributes: (1) an id, (2) children: the two cluster that are merged in the step, (3) distance: the Euclidean distance between the two merged clusters, (4) movie_ids: the ids of the movies in the merged cluster, and (5) full_clustering: all clusters in the current cluster state.

The variable movies contains a list of action films. The most relevant attributes for this task are (1) imdbId: the id of the film, (2) title, (3) genres, and (4) overview: a short description of the film.

Your system should be able to answer the following questions:

  1. How did the clustering evolve? In which region were the sequentially merged clusters close together, based on their Euclidean distance?
  2. Which action films have been clustered together in the different cluster steps? Does it make sense?
  3. What is the distribution of genres in each cluster?
  4. How do the clusters differ in wording?

The data has already been loaded for you in index.js. We are logging it into the console for you to analyze.

Hints

If you want, you can use the provided functionality for creating word clouds. The function is already loaded for you in the document_embedding.js. The function wordcloud(ssvg, word_counts) expects a d3 selection as svg and an array of word counts in the format [[word1, count], [word2, count]...] as word_counts. For tokenization, feel free to use the function documentToWords(text) that receives the raw text as input and returns an array of single words.

If you want to use tfidf, you can use the tfidf(word_array, idf). word_array is an array of word counts. idf is a map of inverse document frequencies that you can calculate using the function inverseDocumentFrequency(grouped_words). The grouped words is an array of arrays of words [[word1, word2, ...], [word3, word 4...]] where each word array represents one group that will be contrasted with the tfidf measure.
